# Toolbox Roadmap

本文档记录墨识从单一 OCR 工具升级为本地识别与快捷工具箱后的功能边界。

## v1 Boundaries

- `OCR`：保留文件、文件夹、图片和 PDF OCR 主流程。
- `截图`：普通区域截图，不进入主窗口页面；从菜单栏入口复用启动时预热的 Swift/AppKit 常驻截图服务并显示原生透明浮层，支持框选、尺寸提示、8 点拖拽调整、选区拖拽移动、矩形、圆形、箭头、画笔、文字、撤销、另存为、关闭、确认复制和 `OCR` 快捷按钮；框选完成后默认不选中标注工具，工具栏以图标展示，按钮悬浮时显示正方形轻量背景，工具栏空白区域不触发重新框选；双击选区仍使用默认截图目录，工具栏另存为由用户选择具体文件位置。
- `截图 OCR`：独立入口，复用原生浮层，框选区域后可直接 OCR；识别文本写入剪贴板，完成后打开轻量 OCR 结果窗口，不进入主窗口页面。
- `OCR 结果窗口`：截图 OCR 完成后打开的独立轻量浮窗页面，左侧截图预览，右侧可编辑识别文本，提供复制、翻译、语言选择和重新识别，不在浮窗内再套弹框层；重新识别使用设置页中的 OCR 引擎，header 图钉可临时固定当前窗口以暂停点击外部自动关闭。
- `翻译`：支持 OpenAI 兼容 Chat Completions API 和火山翻译两个引擎；火山翻译只需要用户填写 AK/SK；设置中按引擎控制启用状态，翻译页选择默认使用的已启用引擎；语言方向自动判断，简体中文译为英文，其他语言译为简体中文；复制原文或译文成功后通过公共 Message 显示轻量提示。
- `剪贴板`：只记录文本历史，不记录图片、文件、富文本，也不做后台自动监听。
- `设置`：包含通用、OCR、翻译、截图、剪贴板、快捷键、后端与扩展、关于八个分类；通用只放界面主题等基础偏好，OCR 分类承载存储目录、输出格式、引擎、语言和 PDF 行为；OCR 默认写入用户文稿目录下的 `墨识/OCR`，截图默认写入用户文稿目录下的 `墨识/Screenshots`。

## Deferred

- 全局快捷键、后台监听剪贴板。
- 贴图、滚动截图、马赛克、序号标注和固定到屏幕。
- 图片剪贴板历史、文件剪贴板历史和富文本历史。
- 本地翻译模型、术语库、划词翻译和沉浸式网页翻译。

## Implementation Notes

- 普通截图和截图 OCR 必须保持两个独立能力；两者都只从菜单栏或全局快捷键触发，不保留主窗口页面入口。
- 普通截图后的 `OCR` 是快捷动作，不改变截图功能本身。
- 截图和截图 OCR 优先复用 `macos-capture` 原生常驻 helper，服务异常时回退一次性 helper；helper 会按 `NSScreenNumber` 匹配 `SCDisplay`，并统一使用 ScreenCaptureKit 的左上角 display 坐标截取选区，避免多屏或 Retina 下输出图片位置偏移；截图 OCR 结果统一进入独立轻量结果窗口，避免两套识别结果交互分叉。
- 后端状态和扩展管理只出现在设置页的 `后端与扩展` 分类中。
